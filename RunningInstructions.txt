===== MUNIN ALERT - HOW TO RUN =====

Requirements:
- Java JDK installed and available in PATH
- Node.js installed and available in PATH

=== OPTION 1: RUN EVERYTHING WITH ONE COMMAND ===

Run the following batch file to start both backend and frontend:
> .\start-all.bat

This will:
1. Check if Java and Node.js are installed
2. Start the backend on http://localhost:8081
3. Start the frontend on http://localhost:3000
4. Open separate command windows for each service

=== OPTION 2: RUN SERVICES SEPARATELY ===

To run the BACKEND only:
1. Open a command prompt in the project root directory
2. Run: .\start-backend.bat
   or
   Run: powershell -File .\start-backend.ps1
3. The backend will start on http://localhost:8081

To run the FRONTEND only:
1. Open a command prompt in the project root directory
2. Run: .\start-frontend.bat
3. The frontend will start on http://localhost:3000

=== MANUAL COMMANDS ===

BACKEND:
1. Navigate to backend directory:
   > cd backend-munin-alert\backend-munin-alert
2. Run using Maven wrapper:
   > mvnw.cmd spring-boot:run

FRONTEND:
1. Navigate to frontend directory:
   > cd frontend\munin-alert-web
2. Install dependencies (if needed):
   > npm install --legacy-peer-deps
3. Start development server:
   > npm start

=== ACCESSING THE APPLICATION ===

- Backend API: http://localhost:8081
- Frontend interface: http://localhost:3000
- Health endpoint: http://localhost:8081/api/public/health

=== STOPPING THE APPLICATION ===

To stop the services, simply close their respective command windows
or press Ctrl+C in each terminal window.
